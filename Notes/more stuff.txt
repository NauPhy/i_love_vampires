AAttackActor (which is a UCLASS) and each of its inheritors will define a USTRUCT in the same .h file (not within their class body) that contains their initialisation data including default values. The master table row struct TM will then include every one of these structs. When you are adding new weapons to the game, you do not need to remember what fields belong to what subclasses- that'll be there already. What you DO need to know is the inheritance path to the subclass you're using, and you may need to override fields in each struct corresponding to a subclass on that path.
This will result in empty/unrelated/unused entries on every row. The only way to avoid that is creating multiple data tables (or reading from json).

Additionally, the USTRUCT defined in the .h file of each subclass will contain 2 USTRUCTS as its data members- _config and _attributes- which will also be defined in that file. So 3 USTRUCTS and 1 UCLASS total.
